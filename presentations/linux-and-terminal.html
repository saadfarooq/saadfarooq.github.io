Basic OS / File system

Linux

  • Unix-compatible free OS
  • Kernel is the basic OS component
  • A "shell" provides an interface to access services of a kernel. http://en.wikipedia.org/wiki/Shell_(computing)

File System Dictory Structure

  • Important concept: EVERYTHING IS A FILE https://help.ubuntu.com/10.04/installation-guide/i386/directory-tree.html

Home Directory

  • Has user files
  • Setting in user directory is usually in "dotfiles"
  • Abbreviated with ~ (tilde)

SHELL COMMANDS

(we use 'bash')

File System Navigation

  • mkdir
  • cd
  • ls
  • rm
  • ./ - execute something in current directory

NOTE: General rule - --help --version usually work with

Permissions

  • Two levels of permissions
    • Owner
    • Group
  • chown, chgrp, chmod
  • Permission are a 9-bit array

Environment

  • env - show current environment
  • echo - output a variable
  • eval - execute a variable
  • PATH - important environment variable
  • which - where a file in path is
  • cat - view a file

Personalizing (non-extreme)

  • alias - make your own shorthand
  • .bashrc - customize your bash startup
  • ln -s - make symboic links to commonly used items
  • man - useful for help (before the internet I guess)

Piping

  • | - pipe the output of one program to the other
  • grep - commonly used for filtering
  • more or less - if info more than one screen
  • > - direct output to a file

Scripting / Automation

Stuff I Missed

  • When an application is run in bash, it makes bash busy
  • To make the app run in background, either :
    • Add an ampersand to the command like 'subl &' to make hte app start in background
    • Press Ctrl + Z at the busy command line and the app will be sent to background
  • jobs - shows which jobs are running
  • fg - brings the job to foreground again (add a job number if there are more than one)
